What is HDL

HDL stands for Hardware Description Language. However, it is not a programming language. A HDL transcript just describe what chips will be used and what signal will be sent to there pins.

Some useful tips

  1. One can never establish a for-loop using HDL. While being execute, HDL runs in a combinatorial logic. That is, the inputs are fixed and unchanging, the output is just a function of the input and is computed ‘instantaneously’. Except for the appearance of time chip, we ignore the issue of time.
  2. We establish an ‘if’ sentence using a multiplexor.
  3. The signal we send to pins will always be ‘undetermined’. That is, it is not allowed to send a ‘1’ or a ‘0’ to a pin in the transcript.Updated in 07-05: In fact ‘false’ stands for signal ‘0’ (1 bit or 16 bits) and ‘true’ stands for signal ‘1’ (1 bit or 16 bits).
  4. The signal we send out of a chip will always ‘stick together’. Although sub-bus is widely used, sentence like:
    And16(a=x[0..15], b=y[0..15], out=out);
    is unacceptable.(Suppose x and y are both output of some chips and are 32-bits).
    But we can instruct like this:
    And16(a=x, b=y, out[0..7]=out1, out[8..15]=out2, out=totaloutput)
