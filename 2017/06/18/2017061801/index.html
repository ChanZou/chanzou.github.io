Docker

Personally, I feel docker as a platform to make the running environment of your application more portable, while keeping light. Simply write a text configuration file and you will have an image that can be brought everywhere. A complicated application may require composing of multiple images. Today we will have a look at what technologies docker uses to created isolated environment and minimize images size.

Namespace

Linux namespaces are a feature of the Linux kernel that isolates system resources of a collection of resources. Currently, Linux implements six different types of namespaces. The purpose of each namespace is to wrap a particular global system resource in an abstraction that makes it appear to the processes within the namespace that they have their own isolated instance of the global resource. This is the technology docker adopts to isolate containers’ processes.
-. Mount namespaces isolate the set of filesystem mount points seen by a group of processes. Thus, processes in different mount namespaces can have different views of the filesystem hierarchy.
-. UTS namespaces isolate two system identifiers—nodename and domain name—returned by the uname() system call, allowing each container to have its own hostname and NIS domain name.
-. IPC namespaces isolate certain interprocess communication (IPC) resources.
-. PID namespaces isolate the process ID number space. Basically making the process PID equals 1.
-. Network namespaces isolate system resources related to network. Each network namespace has its own network devices, IP addresses, IP routing tables, /proc/net directory, port numbers, and so on.
-. User namespaces isolate the user and group ID number spaces.
Inside Linux, clone() can be called to initiate a new process with parameter fed to created an isolated space.

CGroup

Environment isolation is the first step of virtualization. We need to isolate resources on our machine. That is where Linux CGroup becomes convenient. Linux CGroup stands for Linux Control Group and provide functionalities like resources limitation, prioritiation, accounting and process control.
