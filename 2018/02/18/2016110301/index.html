Amazing Feature Engineering Tricks

By Chan Zou

2018-02-18 更新日期:2018-02-18

文章目錄

For long time I regarded feature engineering as combining orginal features using domain knowledge and common sense, then prune it using feature importance generated by random forest and k-features logisitic regression (select k features with highest model accuracy). But one post from kaggle forum taught me a lot and I would like to share what I learnt with you.

Suppose we have features Actions, ActionTypes, ActionDetails, Devices and seconds_elapsed for a browsing session of one website. It is obvious that the last feature is numeric and the others are nominal variables.

We can fill in blanks with “NULL” it is a new level.

Then following expansion in feature space can be conducted for each website user (note that one user can have multiple sessions):

  1. Number of actions taken
  2. Number of unique Actions, ActionTypes, ActionDetails, Devices
  3. Sum, mean, min, max, median, s.d., skewness, kurtosis of seconds_elapsed (here we assume that seconds_elapsed is sampled from a Gaussian distribution. Each user may have a different distribution.)
  4. Entropy of Actions, ActionTypes, ActionDetails, Devices, Secs_elapsed of the first, second, last, second-to-last, and third-to-last Actions. (Brilliant! Some users may have consistent browsing behavior while others do not, this is reflected by information entropy)
  5. Ratios of unique Actions, ActionTypes, ActionDetails, Devices
  6. Ratios of entropies
    Then we cast the sessions frame so each level of Action, ActionType, ActionDetail, and Device had its own column. Two levels are included, once using the count and the second time using the sum of seconds_elapsed.
    One of result columns of cast operation above is: iPhone_deviceSecs. That is, the sum of seconds elapsed of sessions where this user use an iPhone. As you can imagine, we might have lookup_actiontypeSecs as well, as lookup is one action type.
    Furthermore, somtimes we convert count and sum into ratio.

Another creative way of generating ‘helper columns’, one for each search keyword (there are 12 keywords in the whole data set). Find the average of each of features for each keywords, then find for each keyword which features were most different based on standard deviation. For example, say the keywords were [NDF, US, FR, IT] and their averages for “feature_1” are [1.04, 5.40, 0.50, 0.98], then “feature_1” would be one of the features we use in creating a helper column for the US. we find 30 of these columns for each keyword and sum them together and that becomes the “helper” column.

Other features can be genrated with common sense. The merit is enough features are nice complementation of fancy models and are necessary for good results.

Remember that we have “NULL” as a unique value, which means that our feature matrix can be sparse. Construction of feature-feature similarity graph and smooth sparse matrix with it (similar to PageRank) could help with it.
